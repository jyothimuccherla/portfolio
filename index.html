ERP Integration Configuration Guide

Audience: System administrators and integration developers | Tools: MadCap Flare, Zendesk

This guide provides configuration procedures for connecting the platform to enterprise resource planning (ERP) systems. Each ERP system requires specific prerequisites, authentication methods, and connection parameters.

Note: All connections support optional integration through a secure gateway agent for on-premises deployments.

SAP S/4 HANA

SAP S/4 HANA integration uses the Operational Data Provisioning (ODP) framework for efficient data extraction and delta processing.

Source System Prerequisites

Step 1: Enable ODP Framework

  1. Use Transaction SU01 to create RFC user
  2. Use Transaction SICF to activate ODP services
  3. Navigate to path: /sap/opu/odata/sap/
  4. Verify ODP_EXTRACTION service is active

Step 2: Configure Operational Data Provisioning

  1. Use Transaction RODPS_OS_EXPOSE to expose data sources
  2. Select data sources to make available
  3. Activate for operational data provisioning
  4. Test using transaction RODPS_REPL_TEST

Step 3: User Authorizations

Authorization ObjectDescription
S_RFCExecute RFC functions
S_TABU_DISTable display authorization
S_BTCH_JOBBackground job scheduling
S_ODPEODP extraction authorization

Connection Configuration

Application Server: [SAP system hostname/IP]

Client: [3-digit client, e.g., 100]
Protocol: HTTP or HTTPS

SAP SuccessFactors

SuccessFactors integration requires OAuth 2.0 authentication with SAML bearer assertion.

Prerequisites

  1. Enable OData API Access in Admin Center
  2. Create dedicated integration user
  3. Configure OAuth 2.0 Client Application
  4. Generate X.509 certificate

Connection Configuration

IDP: https://[tenant_id]/odata/v2
Authentication Type: saml2-Bearer
IDP URL: https://[tenant_id]/oauth
Parameters:
  client_id: [client_id]
  user_id: [user_id]
  token_url: [token_url]

Bank Account Reconciliation Workflow

Audience: Finance managers, accounting staff | Tools: MadCap Flare, Zendesk

Bank account reconciliations use AI-powered automatic matching to reconcile general ledger transactions against bank statement data. The system automatically parses uploaded statements and matches transactions, eliminating 80-90% of manual reconciliation work.

Step 1: Open Bank Account Reconciliation

Begin by navigating to the bank account you want to reconcile.

  1. Navigate to the Reconciliations module
  2. Locate the bank account (identified by bank account number)
  3. Click on the account row to open Reconciliation Details page

Step 2: Upload Bank Statements

Upload bank statement files to initiate AI-powered automatic transaction matching.

To Upload Bank Statements

  1. Click the Upload button on the Reconciliation Details page
  2. In the upload dialog: click Browse Files or Drop files in upload area
  3. Select your bank statement file (Excel, CSV, PDF, or image formats)
  4. Verify the file contains: account number, statement period, transaction amounts, descriptions
  5. Ensure transactions match the selected accounting period
  6. Click Open to upload the file
  7. Click Upload to upload the file

Monitor File Processing

StatusIndicatorMeaning
ProcessingGreen spinnerFile is being parsed
AttachedGreen checkmarkFile successfully processed
ErrorRed iconProcessing failed: wrong period data, corrupted file, or missing field

After successful upload:

  • System automatically parses the file
  • AI engine performs automatic matching
  • Matched Results tab populates with matched pairs
  • Unmatched Items tab shows remaining transactions
  • KPI cards update with current totals

Step 3: Set Support Balance

Manually enter the closing balance when not included in the uploaded bank statement.

Reconciliation Scenarios

ScenarioGL vs Support BalanceRequired Action
BalancedGL Balance = Support BalanceSubmit for review with no unmatched items
Variance ExistsGL Balance ‚â† Support BalanceAdd unmatched items or comments to explain difference

Step 4: Review Matched Transactions

Verify the accuracy of AI-matched transaction pairs.

FieldWhat to Check
Match IDUnique identifier with color coding
DatesGL date corresponds to bank date
AmountsGL amount equals bank amount exactly
DescriptionsGL description relates to bank description
Trading PartnersGL and bank trading partners match

Unmatching Incorrect Pairs

  1. Locate the incorrect match
  2. Click the Unmatch icon on the right side
  3. Both transactions move to Unmatched Items tab
Restriction: You can only unmatch transactions in Preparation stage. Once submitted, transactions are locked.

Step 5: Manage Unmatched Transactions

Review transactions requiring manual matching using side-by-side comparison view.

Step 6: Manually Match Transactions

Create manual matches for transactions the AI engine could not pair automatically.

  1. Find a GL entry and bank line with the same amount
  2. Select the checkboxes next to both transactions
  3. Verify the selection summary bar shows matching totals
  4. Click Match Selected to link the transactions
  5. (Optional) Add a comment to explain unusual aspects

Step 7: Document Unmatched Transactions

Add explanatory comments for transactions that cannot be matched in current period.

Common Unmatched Items

  • Outstanding checks not yet cleared by the bank
  • Deposits in transit
  • Bank fees or charges not yet recorded in GL
  • Timing differences between systems

Documentation Process

  1. Navigate to the Unmatched Items tab
  2. Select the unmatched transaction(s)
  3. Click Add Comment
  4. Enter clear explanation including reason, expected resolution date, and reference numbers

Step 8: Submit Reconciliation for Review

Complete the reconciliation process by submitting for management review.

Pre-Submission Checklist

  • ‚úì All possible matches have been identified and matched
  • ‚úì Remaining variances are documented with clear comments
  • ‚úì KPI cards show acceptable reconciliation totals
  • ‚úì Support balance matches GL balance (or variances are explained)

Submission Process

  1. Review the reconciliation summary in KPI cards
  2. Click Submit for Review
  3. (Optional) Add comments for reviewers
  4. Click Confirm

Tips for Efficient Reconciliation

Before uploading statements:

  • Ensure statement covers correct accounting period
  • Verify statement format is readable
  • Check all transaction details are visible

During matching:

  • Start with exact amount matches - these are most reliable
  • Review AI-suggested matches before accepting
  • Document unusual items immediately
